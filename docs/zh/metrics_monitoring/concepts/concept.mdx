# 核心概念

## 流量

数据在网络、系统或应用程序中的流动。这些数据可以包括用户生成的请求、服务响应或数据包，涵盖了信息技术和网络环境中各种类型的数据流。

## 监控指标

用于衡量和评估系统、网络或服务性能和健康状况的定量数据。这些指标是IT运营和系统管理中的重要元素，用于持续监测系统的健康、性能和资源利用情况，以确保服务稳定并优化用户体验。

## API 监控

持续检查和评估微服务 API（应用程序编程接口）性能和响应能力的过程，观察 API 是否符合设定的性能标准。

## JVM 监控

跟踪和分析运行 Java 应用程序的 JVM 实例的性能和资源使用情况的过程。JVM 是执行 Java 程序的环境，使 Java 程序能够在各种硬件和操作系统平台上运行，而无需重新编译。JVM 监控对确保 Java 应用程序高效和稳定的运行至关重要。

## RPS

每秒请求数的缩写，是衡量系统、服务器或应用程序性能的关键指标，表示系统每秒可以处理多少请求。

## TP50

也称为 P50 或中位数。该性能指标衡量数据集的中点。例如，在网络响应时间或服务器处理时间的上下文中，**TP50** 表示 50% 的请求响应时间低于或等于该值，代表中位响应时间。性能监测和统计分析中常用的分位数包括 TP90 或 TP99，分别代表第 90 和第 99 个分位数。这些指标帮助管理员理解系统在不同压力或负载条件下的性能。
