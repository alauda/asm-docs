# 核心概念

## 流量

数据通过网络、系统或应用程序流动。这些数据可能包括用户生成的请求、服务响应或数据包，涵盖信息技术和网络环境中的各种类型的数据流。

## 监控指标

用于测量和评估系统、网络或服务性能和健康状况的定量数据。这些指标是 IT 操作和系统管理中的基本元素，用于持续监控系统的健康、性能和资源利用，以确保服务的稳定性并优化用户体验。

## API 监控

持续检查和评估微服务 API（应用程序编程接口）性能和响应能力的过程，观察 API 是否满足设定的性能标准。

## JVM 监控

跟踪和分析运行 Java 应用程序的 JVM 实例的性能和资源使用情况的过程。JVM 是执行 Java 程序的环境，使得 Java 程序可以在各种硬件和操作系统平台上运行，而无需重新编译。JVM 监控对于确保 Java 应用程序的高效和稳定运行至关重要。

## RPS

表示每秒请求数，这是一个衡量系统、服务器或应用程序性能的关键指标，表示系统每秒能够处理多少个请求。

## TP50

也称为 P50 或中位数。该性能指标测量数据集的中点。例如，在网络响应时间或服务器处理时间的上下文中，**TP50** 表示 50% 的请求响应时间低于或等于该值，代表中位响应时间。在性能监控和统计分析中常用的百分位数包括 TP90 或 TP99，分别代表第 90 和第 99 百分位。这些指标帮助管理员了解系统在不同压力或负载条件下的表现。
