# OpenTelemetry 流量指标

该平台提供丰富的流量指标数据，使用户能够从多个维度分析服务流量的质量。

## 服务流量监控

### 查看服务流量质量

**前提条件**

该服务已注入 Java Agent，请参考 [添加服务](../../microservice/add-service.md#add-opentelemetry-service) 进行实现。

**步骤**

1. 在左侧导航栏中，点击 **监控**。

2. 选择要查看监控数据的服务和时间范围。

   **注意**：查询的时间范围受 Prometheus 监控数据的保留期限制。例如，如果监控数据的保留期为 7 天，而设置的时间范围为过去 30 天，则统计数据仅覆盖 7 天。

3. 点击相应的选项卡以查看该服务的流量监控数据和 JVM 监控数据。

### 常规操作

- **刷新数据**：当前页面的监控统计数据在页面打开时仅会自动刷新一次。要再次刷新，可以使用以下两种方法：

  - 手动刷新：点击 <img src="../assets/metric_refresh.png" style={{display: "inline", verticalAlign: "middle"}} /> 在页面右下角手动刷新数据。

  - 设置自动刷新（默认关闭）：点击 <img src="../assets/metric_auto_refresh.png" style={{display: "inline", verticalAlign: "middle"}} /> 设置自动刷新数据的时间间隔。

- **查看/设置图例**：点击图表右上角的 <img src="../assets/legend.png" style={{display: "inline", verticalAlign: "middle"}} /> 展开监控图表中的图例。点击图例可以隐藏/显示图表中的相应曲线。

- **查看大图监控图表**：点击图表右上角的 <img src="../assets/big_chart.png" style={{display: "inline", verticalAlign: "middle"}} /> 在弹出对话框中查看放大后更详细的监控图表。

### 监控数据说明

| 监控指标                         | 说明                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|----------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **平均响应时间**                 | 查询时间范围内的平均响应时间（总响应时间/总响应次数）。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **平均进/出 RPS**                | 查询时间范围内的进/出 RPS（每秒请求数）的平均值。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **响应时间**                     | 服务之间或服务内部的响应时间，显示为平均值、TP 50、TP 95、TP 99。<br />TP（顶部百分位）xx 表示 xx 百分比的网络请求所需的最小持续时间，通常用于系统性能监控场景。<br />鼠标悬停在曲线上可以查看特定时段的响应时间。                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **进/出 RPS**                    | 查询时间范围内的总进/出流量 RPS 和错误进/出流量 RPS。<br />**RPS = 查询时间内的请求次数 / 查询时长（秒）**。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **进/出流量**                    | 查询时间范围内的总进/出请求量，以及按 HTTP 返回码（正常/2xx、3xx、4xx、5xx）分类的流量比例。<br />鼠标悬停在柱状图上可以查看每个类别的流量比例。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **客户端流量比较**               | **客户端**指向当前服务（上游服务）发起请求的客户端服务（下游服务）。**客户端流量比较**将比较并显示调用当前服务的不同客户端的 **响应时间**、**进 RPS** 和进 **RPS 错误率**（Error RPS/RPS）。<br /><br />**说明**:<br />- **未知客户端**指的是在服务网格中调用当前服务的客户集合（HTTP 协议），其流量不受 OpenTelemetry 管理。<br />- 如果客户端属于当前命名空间，点击客户端名称旁边的 **跟踪** 将重定向到跟踪页面。
