# JVM 监控指标

JVM 监控页面显示由 OpenTelemetry Java Agent 收集的 JVM（Java 虚拟机）指标数据。指标包括垃圾回收（GC）、堆内存、非堆内存、JVM 线程、Java 类等。您可以使用监控图表中呈现的实时监控数据进行性能分析，快速识别内存泄漏、线程异常和其他问题。

为了满足不同的监控指标分析场景，还支持以下功能：

- 当服务有多个 Pods 时，您可以并排比较两个 Pods 的监控数据。
- 灵活地筛选要显示的监控指标。

## 先决条件

服务已注入 Java Agent，具体实现在于参考 [添加服务](../../microservice/todo-add-otel-service.md)。

## 步骤

1. 在左侧导航栏中，点击 **监控**。

2. 选择您想查看监控数据的服务和时间范围。

   **注**：查询时间范围受 Prometheus 监控数据的保留期限制。例如，如果监控数据的保留期为 7 天，而设置的时间范围为过去 30 天，则统计数据仅涵盖 7 天。

3. 点击 **JVM 监控** 标签以查看该服务 Pod 的 JVM 监控数据。

   **提示**：当服务有多个 Pods 时，点击查看 **监控数据比较**，以分析和比较 2 个 Pods 的监控数据。

   监控图表显示在查询时间范围内所选服务 Pod 的 JVM 监控指标趋势统计。统计数据包括各种类的最大、分配和使用内存；CPU 使用率；GC 次数和持续时间；活动线程数；加载/卸载类的数量等。

## JVM 监控图表说明

监控图表显示在查询时间范围内所选服务 Pod 的 JVM 监控指标趋势统计。监控指标的说明如下表所示。

**提示**：在查看数据时，您可以参考 [常规操作](./otel_traffic_metrics.mdx#regular-operations) 刷新监控数据，隐藏与图例对应的数据，或以较大格式查看监控图表。

| 监控指标                       | 描述                                                                                                                                                               |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **年轻代 GC 次数/持续时间**     | Pod 在年轻代的垃圾回收次数及所耗费的时间。                                                                                                                       |
| **老年代 GC 次数/持续时间**     | Pod 在老年代的垃圾回收次数及所耗费的时间。                                                                                                                       |
| **系统/Pod CPU 使用率**        | **系统**：Pod 系统内的 CPU 使用率（已使用的 CPU 数量 / 分配给 Pod 的总 CPU 数量）<br />**Pod**：Pod 的 CPU 使用率（已使用的 CPU 数量 / 分配给 Pod 的总 CPU 数量） |
| **堆内存**                     | Pod 的可用堆内存和堆内存使用情况统计，包括：**峰值可用堆内存**、**当前可用堆内存**、**已使用堆内存**。                                                         |
| **堆内存详情**                 | Pod 中各个堆分区的内存使用情况统计，包括：**年轻代 Eden 区**、**老年代区**、**年轻代 Survivor 区**、**总已使用堆内存**。                                           |
| **非堆内存**                   | Pod 的可用非堆内存和非堆内存使用情况统计，包括：**峰值可用非堆内存**、**当前可用非堆内存**、**已使用非堆内存**。                                              |
| **元空间内存**                 | Pod 的可用元空间内存和元空间内存使用情况统计，包括：**总元空间大小**、**已使用元空间大小**。                                                                    |
| **直接缓冲区内存**             | Pod 的可用直接缓冲区内存和直接缓冲区内存使用情况统计，包括：**总直接缓冲区大小**、**已使用直接缓冲区大小**。                                                  |
| **守护线程**                   | Pod 中的守护线程数量。                                                                                                                                          |
| **活动线程**                   | Pod 中的活动线程数量。                                                                                                                                          |
| **JAVA 类使用情况**            | Pod 加载的 Java 类数量。                                                                                                                                       |
| **卸载类计数**                 | Pod 卸载的 Java 类数量。                                                                                                                                       |
