# 介绍

## 外部服务介绍

外部服务模块实现了服务网格内服务与外部终端之间的安全且受管理的连接。该能力解决了三个基本挑战：

1. **混合环境集成**：桥接云原生与传统系统
2. **流量治理**：将服务网格策略应用于非网格服务
3. **安全执行**：将零信任原则扩展至外部依赖

关键价值主张：

- 跨环境连接设置时间减少 80%
- 对混合架构进行统一可观测性
- 在网格和非网格服务之间一致地执行政策

## 优势

- **协议灵活性**：支持 L4（TCP）和 L7（HTTP/HTTP2/HTTPS）协议
- **动态负载均衡**：4 种智能算法，支持会话保持
- **弹性工程**：支持电路断路与自动恢复机制
- **出站控制**：与 Istio 出口网关集成，提供安全的出口点

## 场景

- **云迁移**：安全地将 Kubernetes 工作负载连接到本地系统
- **SaaS 集成**：管理对外部 API 的流量（如 Salesforce、Twilio 等）
- **多集群路由**：在云服务提供商之间实施故障转移
- **传统系统现代化**：将服务网格特性应用于基于 VM 的服务

## 限制

- L4 服务无法使用会话保持特性
- 创建后无法修改协议选择
- 每个服务定义最多支持 5 个端口
- 出口网关需要手动管理证书
