# 如何为集群设置区域

:::tip
通常，公共云集群带有区域属性。本文档说明如何为平台上自建集群或连接到平台的私有部署集群设置区域属性。
:::

- 如果需要为**待添加**到网格的集群设置区域，请参阅 [为集群设置区域]()。

- 如果需要为**已添加**到网格的集群设置区域，请按照以下步骤操作：

  1. 执行操作 [为集群设置区域]()。

  2. 登录到集群的任意控制节点，并执行以下命令以重启在集群中部署的 Istiod 组件：

     ```sh
     # 如果 Istiod 的版本号为 1.18，则 Istiod 部署的默认名称为“istiod-1-18”；Istiod 默认为在 istio-system 命名空间中部署。
     kubectl rollout restart deployment <Istiod Deployment Name> --namespace=<Istiod Namespace>
     ```

  3. 执行以下命令以重启集群中所有已注入 Sidecar 的服务：

     ```sh
     kubectl rollout restart deployment <Service Deployment Name> --namespace=<Service Namespace>
     ```
