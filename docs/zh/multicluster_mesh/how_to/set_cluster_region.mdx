# 如何为集群设置区域

:::tip
通常情况下，公有云集群会附带区域属性。本文件说明了如何为平台上的自建集群或连接到平台的私有部署集群设置区域属性。
:::

- 如果您需要为 **待添加到服务网格的集群** 设置区域，请参考 [如何为集群设置区域]()。

- 如果您需要为 **已添加到服务网格的集群** 设置区域，请按照以下步骤进行：

  1. 执行操作 [如何为集群设置区域]()。

  2. 登录到集群的任意控制节点，并执行以下命令以重启部署在集群中的 Istiod 组件：

     ```sh
     # 如果 Istiod 的版本号为 1.18，则 Istiod 部署的默认名称为 “istiod-1-18”；Istiod 默认部署在 istio-system 命名空间下。
     kubectl rollout restart deployment <Istiod Deployment Name> --namespace=<Istiod Namespace>
     ```

  3. 执行以下命令以重启集群中所有注入 Sidecar 的服务：

     ```sh
     kubectl rollout restart deployment <Service Deployment Name> --namespace=<Service Namespace>
     ```
