# 配置和验证跨区域集群权重分配

本文档说明了如何为基于多集群服务网格的应用程序实现跨集群权重分配，以 Bookinfo 应用程序为例。

## 步骤

1. 在左侧导航栏中，点击 **服务网格** > **网格**。

2. 点击要配置的 ***服务网格名称***。

3. 在 **网格策略** 选项卡中，点击集群 `c1` 右侧的 **创建策略** > **区域负载均衡**。

4. 选择 **权重分配** 作为 **策略类型**，并根据以下说明配置相关参数。

   | 参数         | 描述                                                                                                                                                            |
   | ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **区域**     | 当前服务网格下能够提供跨区域负载均衡的区域。**当前区域** 是当前集群所属的区域。                                                                              |
   | **集群**     | 区域内的集群。当同一区域内有多个集群时，流量将在该区域内的集群之间均匀分配。                                                                                    |
   | **权重**     | 负载流量的百分比。<br />**注意**：所有区域的总权重必须等于 100。                                                                                              |

   **示例**：将集群 `c1` 所在区域的权重设置为 80%，将集群 `c2` 所在区域的权重设置为 20%。

5. 点击 **创建**。

### 验证方法

请参考 [访问验证](../how_to/deploy_bookinfo.mdx#access-verification) 方法，持续访问基于集群 `c1` 部署的网关所创建的路由配置（`外部访问地址`）的 productpage 服务。

您可以前往服务网格平台的 **监控** 页面，并在 **集群流量对比** 面板中比较各集群中 productpage 服务的监控数据。如果流量监控数据与权重比例相对应，则表明跨集群权重分配有效。
