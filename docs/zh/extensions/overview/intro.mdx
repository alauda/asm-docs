# 介绍

## Istio 扩展机制简介

Istio 提供了可扩展性功能，允许与 Envoy 数据平面进行深入集成，同时维护其控制平面抽象的完整性。这些扩展机制使平台运营者能够实现超出标准 Istio API 范围的高级自定义场景，特别通过 **EnvoyFilter** 资源——当前 Istio 实现中的扩展接口。

:::tip 核心设计原则
Istio 扩展遵循 *受控自定义* 的原则，使得在保持与 Istio 声明式配置模型兼容的前提下，能够进行代理层级的修改。这在灵活性与操作安全之间保持了平衡。
:::

## 关键扩展能力

### EnvoyFilter

作为 Istio 的核心扩展机制，EnvoyFilter 允许通过以下关键操作直接操作 Envoy 代理配置：

- **过滤器注入**\
  将自定义 L4/L7 过滤器插入到 Envoy 的过滤器链中
- **协议自定义**\
  通过自定义编码器实现非标准协议处理
- **运行时参数调优**\
  为特定工作负载覆盖默认的 Envoy 设置
- **动态配置**\
  在运行时修改监听器、集群和路由
