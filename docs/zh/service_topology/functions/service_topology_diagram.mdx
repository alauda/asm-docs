# 服务拓扑图

本文旨在帮助您快速掌握查看服务拓扑的过程。

## 先决条件

要查看服务拓扑，您需要满足以下任一条件：

- 对于 Istio 服务：确保已为服务注入 Sidecar。详细信息请参见 [添加 ServiceMesh 服务](../../microservice/add-service.md#add-servicemesh-service)。
- 对于 OpenTelemetry 服务：确保已为服务注入 OpenTelemetry Java Agent。详细信息请参见 [添加 OpenTelemetry 服务](../../microservice/add-service.md#add-opentelemetry-service)。

## 快速开始

在左侧导航栏中，点击 **服务拓扑**。

## 基本页面展示

页面清晰地展示了服务之间的调用关系，每个节点和连接线通过不同的颜色加以区分。

- 连接线表示服务之间的调用，其颜色根据调用的成功率进行调整。灰色线条表示在选定时间段内此调用没有流量。
- 节点代表单个服务，其颜色由服务 Pods 的比例决定。

有关节点图标的详细说明、节点和连接线的颜色阈值及其含义，请参阅拓扑图左上角提供的图例。

当某个服务（及其 Deployment、Pod）或与报警规则相关的入口网关触发报警时，拓扑图将以 **报警** (<img src="../assets/firing.png" style={{display: "inline", verticalAlign: "middle"}} />) 标记该服务或入口网关。点击服务的 <img src="../assets/firing.png" style={{display: "inline", verticalAlign: "middle"}} /> 标记将重定向您到 **实时报警** 页面，以查看报警规则的详细信息。

默认情况下，拓扑图每 5 秒自动刷新一次。您可以手动刷新或设置自动刷新间隔：<br />**手动刷新**：点击拓扑图右下角的 <img src="../assets/metric_refresh.png" style={{display: "inline", verticalAlign: "middle"}} /> 手动刷新数据；<br />**设置自动刷新间隔**：点击右下角 <img src="../assets/metric_refresh.png" style={{display: "inline", verticalAlign: "middle"}} /> 旁边的间隔时间以设置自动刷新间隔。

## 查看 Istio 跨集群拓扑

如果当前项目与多个由服务网格管理的集群关联，选择 **跨集群拓扑** 可查看在当前服务网格和项目的所有命名空间中注入了 Sidecar 的服务和入口网关的调用关系拓扑图，前提是您具备相应的账户权限。<br /><br />**注意**：可查看的范围取决于用户的账户数据权限；仅相同命名空间内的服务支持查看 Trace。例如：集群 A 和 B 由当前服务网格管理，命名空间 an1 和 bn1 分别存在于集群 A 和 B 下，而您具有 an1 和 bn1 的查看权限。当您进入 an1 以查看服务拓扑时，默认情况下，您只能查看 an1 命名空间内的所有服务的调用拓扑图；当您选择 **所有集群** 时，您可以查看 an1 和 bn1 命名空间内所有服务的调用拓扑图，包括 an1 中服务调用 bn1 中服务的拓扑图。

## 查看指定命名空间下服务的调用拓扑

点击并选择拓扑图上方的 **命名空间** 下拉框。您可以查看当前集群中指定命名空间内服务的调用拓扑。默认设置为所有命名空间，显示集群中所有管理命名空间的调用关系。

## 查看本地拓扑图

您可以 **双击** 节点（除“主机未知”的中间件节点外）以展开以该节点为中心的本地拓扑。这允许您快速查看目标节点的上下游调用关系中所有节点的状态和数据，帮助您更专注于分析特定服务的调用情况。

## 查看详细信息

您可以 **点击** 节点/连接线以展开右侧详细面板，在这里可以进行多项操作。

### 查看 Trace

当您点击 **节点** 或 **连接线** 时，详细面板会显示 **查看 Trace**，您可以前往 **Trace** 页面查看当前 **服务** 或 **调用关系** 的 Trace 数据。

- 点击节点 > 查看 Trace：此操作查询在拓扑图中设置的当前时间范围内所选服务报告的所有 span 数据。例如，如果您点击一个名为“client”的服务节点并点击 **查看 Trace**，系统将搜索当前时间范围内属于名为 client 的服务的所有 spans。
- 点击连接线 > 查看 Trace：此操作查询在拓扑图中设置的当前时间范围内与特定调用关系对应的 span 数据。例如，如果您点击“client”服务与“server”之间的连接线并点击 **查看 Trace**，系统将搜索当前时间范围内“client”是调用方而“server”是被调用方的所有 spans。

**注意**：查看 Trace 功能支持根据您的服务网格实现的不同类型的服务：

- 对于 Istio：仅支持 **ServiceMesh 服务**、**入口网关** 和 **出口网关** 节点。
- 对于 OpenTelemetry：支持 **OpenTelemetry 服务** 和 **中间件** 节点（如数据库和消息队列）。

如果您点击不支持的节点类型或包含不支持节点的连接线，将无法导航到 Trace 页面。

### 查看服务流量监控

当您点击 **ServiceMesh 服务**、**OpenTelemetry 服务**、**入口网关**或 **出口网关** 节点时，在详细面板中点击 **查看流量监控** 可导航至 **监控** 页面，以查看当前服务的详细监控数据。

### 查看流量信息

**注意**：

- 流量信息面板仅汇总由 Sidecar 管理的流量数据，不包括与未知客户端的通信流量数据（未由 Sidecar 管理的流量）。您可以访问 **服务详细信息** 或 **监控** 页面以查看完整的流量数据。

- 当平台收集流量数据时，如果客户端已注入 Sidecar，则流量采样率与在服务网格中配置的调用链采样率相同；否则，采样率为 100%。

- 流量数据的查询时间范围取决于与服务网格集成的监控系统的数据保留配置。例如，如果监控系统（如 Prometheus）配置为保留过去 7 天的数据，则在查询时间范围超过 7 天时，系统只能显示最近 7 天的流量数据。

当您点击 **节点** 或 **连接线** 时，您可以在详细面板中查看当前 **服务** 或 **调用关系** 的流量图表。不同协议的图表说明如下：

#### HTTP/HTTP2/gRPC 协议

| 参数                           | 描述                                                                                                                                                                                                                                                                                                                                        |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **收入/支出流量**              | 查询时间范围内的总收入/支出请求量，以及按 HTTP 状态码（正常/2xx、3xx、4xx、5xx）的流量比例。<br />将鼠标悬停在柱状图上以查看每个类别的流量比例。                                                                                                                                                                           |
| **收入/支出 RPS**              | 查询时间范围内的总收入/支出流量 RPS（每秒请求数）及错误的收入/支出流量 RPS。<br />**RPS = 查询时间内的请求数 / 查询时长（s）**                                                                                                                                                                                               |
| **流量**                       | 查询时间范围内服务之间的请求总数，以及按 HTTP 状态码（正常/2xx、3xx、4xx、5xx）、NR（无响应）的请求数量和比例。<br />将鼠标悬停在柱状图上以查看每个类别的请求数量和比例。                                                                                                                                                     |
| **RPS**                       | 查询时间范围内的总流量 RPS（每秒请求数）及错误流量 RPS。                                                                                                                                                                                                                                                                          |
| **响应时间**                   | 服务之间或服务内部的请求响应时间，按平均值、TP 50、TP 95、TP 99 显示。<br />TP（Top Percentile）xx 表示满足 xx 百分比网络请求所需的最小时长，通常用于系统性能监控场景。<br />将鼠标悬停在曲线图上以查看特定时间段的响应时间。                                                                                               |

#### TCP 协议

| 参数                           | 描述                                                                                                                                                                                                                                                                    |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **收入/支出流量**              | 查询时间范围内的总收入/支出流量大小（以字节为单位）。                                                                                                                                                                                                                   |
| **TCP 连接数**                | 连接的总数。<br />**错误率 = 失败连接数 / 总连接数**<br />**成功率 = 成功连接数 / 总连接数**<br />将鼠标悬停在柱状图上以查看不同类别的连接数。                                                                                                               |
| **流量**                       | 查询时间范围内服务之间的流量总大小（以字节为单位）。                                                                                                                                                                                                                   |
| **收入/支出（Bps）**          | 收入/支出服务网络流量的字节传输速率（字节每秒）。                                                                                                                                                                                                                           |

#### **中间件信息**

当您点击连接一个 **OpenTelemetry 服务** 与 **中间件服务** 的 **连接线** 时，您可以查看以下信息：

**节点信息**

| 类型              | 描述                                                                                                                                                                   |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **数据库**        | 类型：数据库类型，如 MySQL、Oracle 等。<br />主机：提供数据库主机地址<br />详情：提供数据库用户名、数据库名称及其他信息 |
| **消息队列**      | 类型：消息队列类型，如 RabbitMQ、Kafka 等。                                                                                                                         |

**注意**：有关支持的中间件类型的完整列表，请参见 [节点类型](../concepts/insight_node_types.mdx#opentelemetry-node-types)。

**流量信息**

指在查询时间范围内 OpenTelemetry 服务对中间件发送的请求总数。
