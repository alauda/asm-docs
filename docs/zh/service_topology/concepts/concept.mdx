# 核心概念

## 服务节点

**服务节点**是在系统或全局调用关系中的独立单元或部分，代表系统内的特定功能组件。例如，这些节点可以包括微服务、网关、消息队列、数据库等。您可以查看支持的[node types](./insight_node_types.mdx)。

## 连接线

表示服务之间的通信路径，例如 API 调用或直接数据流。当“连接线从服务 A 到服务 B”时，意味着服务 A 依赖服务 B 完成其操作。

## 依赖

指服务在执行其功能时所依赖的其他服务。这种**依赖**可以采取多种形式，其中服务调用关系最为常见。例如，当“服务 A 依赖于服务 B”时，意味着服务 A 调用服务 B。
