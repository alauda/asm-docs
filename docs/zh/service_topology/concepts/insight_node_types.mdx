# 节点类型的深入理解

:::note

在拓扑图中，如果识别到的服务调用关系不属于上述明确分类，将用 **未知服务** 节点图标表示，以确保拓扑调用关系的完整性和清晰度。此表示方式帮助用户全面了解整个服务网络的结构。

:::

## Istio 节点类型

在服务网格的管理下，目前平台支持以下节点类型：

1. **ServiceMesh 服务**

2. **入口网关**

3. **出口网关**

4. **外部服务（ServiceEntry）**

5. **PassthroughCluster** - 更多信息，请参阅 [官方文档](https://istio.io/v1.20/blog/2019/monitoring-external-service-traffic/)。

6. **BlackHoleCluster** - 更多信息，请参阅 [官方文档](https://istio.io/v1.20/blog/2019/monitoring-external-service-traffic/)。

## OpenTelemetry 节点类型

在 OpenTelemetry 的管理下，目前平台支持以下节点类型：

1. **OpenTelemetry 服务**

2. **中间件**

- **数据库**
  - Oracle
  - Redis
  - MySQL
  - PostgreSQL
- **消息队列**
  - Kafka
  - RabbitMQ
  - RocketMQ
