# 节点类型的深入理解

:::note

在拓扑图中，如果识别出一个服务调用关系，该关系不属于上述明确分类，它将被表示为一个 **未知服务** 节点图标，以确保拓扑调用关系的完整性和清晰度。这种表示方式有助于用户全面了解整个服务网络的结构。

:::

## Istio 节点类型

在服务网格管理下，当前平台支持以下节点类型：

1. **服务网格服务**

2. **入口网关**

3. **出口网关**

4. **外部服务 (ServiceEntry)**

5. **PassthroughCluster** - 更多信息，请参考 [官方文档](https://istio.io/v1.20/blog/2019/monitoring-external-service-traffic/)。

6. **BlackHoleCluster** - 更多信息，请参考 [官方文档](https://istio.io/v1.20/blog/2019/monitoring-external-service-traffic/)。

## OpenTelemetry 节点类型

在 OpenTelemetry 管理下，当前平台支持以下节点类型：

1. **OpenTelemetry 服务**

2. **中间件**

- **数据库**
  - Oracle
  - Redis
  - MySQL
  - PostgreSQL
- **消息队列**
  - Kafka
  - RabbitMQ
  - RocketMQ
