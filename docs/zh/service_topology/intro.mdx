# 介绍

服务拓扑是指在分布式系统或服务网络中，服务之间的布局和相互连接结构。它主要用于描述服务组件如何组织、相互通信以及它们的依赖关系如何配置。

平台提供的服务拓扑图直观地展示了两种调用关系：Istio 服务网格组件（包括服务、网关和外部服务（ServiceEntry））以及 OpenTelemetry 监控的组件（包括 Kafka、PostgreSQL、MySQL、Oracle、Redis 等各种中间件服务）。

通过拓扑图，您可以了解服务之间的交互，快速定位错误发生的位置，以及如何优化这些交互以提高系统的效率和可靠性。简单的操作（点击节点或拓扑线）可以让您查看与特定服务或拓扑线相关的服务之间的流量信息。

当服务触发警报时，拓扑图中的相关服务节点将带有显眼的特殊标签，允许操作人员通过拓扑图快速识别和理解问题。此外，为 Istio 服务配置的安全措施和熔断策略也会在拓扑图中用特殊标记显示，提供服务状态的全面视图。
