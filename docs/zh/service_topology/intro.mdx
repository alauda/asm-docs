# 介绍

服务拓扑（Service Topology）指的是在分布式系统或服务网络中，服务之间的布局和相互连接结构，主要用于描述服务组件如何组织、相互通信以及它们的依赖关系是如何配置的。

平台提供的服务拓扑图直观地展示了两种类型的调用关系：Istio 服务网格组件（包括服务、网关和外部服务（ServiceEntry））与 OpenTelemetry 监控的组件（包括 Kafka、PostgreSQL、MySQL、Oracle、Redis 等各种中间件服务）。

通过拓扑图，您可以了解服务之间的交互，快速定位错误发生的位置，以及如何优化这些交互以提升系统的效率和可靠性。简单的操作（点击节点或拓扑线）可以让您查看与特定服务或拓扑线相关的流量信息。

当某个服务触发警报时，拓扑图中的相关服务节点将被标记上显眼的特殊标签，便于运维人员通过拓扑图快速识别和理解问题。此外，针对 Istio 服务配置的安全措施与熔断器策略也会在拓扑图中以特殊标记显示，从而提供服务状态的全面视图。
