服务网格是基于 Istio 的 PaaS 平台，为应用程序和微服务提供云原生解决方案。

该平台提供一键基础设施部署和升级能力、可视化服务治理、高效的应用性能管理以及服务网关管理的高可用性与性能。它为正在进行数字化转型的企业提供全面、稳定、可靠和开放的微服务解决方案，帮助提高服务治理效率，降低微服务框架的维护成本，同时减少开发者对框架的依赖，使企业能够专注于业务发展，持续提升核心竞争力。

## 服务网格的全生命周期管理

在多集群环境中，管理服务网格尤其复杂。服务网格的全生命周期管理不仅涉及部署、更新和删除服务网格，还包括实时监控其运行状态，使运维人员能够及时发现异常并排除故障，从而确保对业务运作的持续和稳定支持。

| 特性                                               | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **多主集群服务网格**                               | 多主集群架构的服务网格管理特性可视化了微服务治理平台组件的部署过程。                                                                                                                                                                                                                                                                                                                                                                                         |
| **服务网格的可视化部署与管理**                     | - 管理员可以使用可视化表单轻松部署每个需要服务治理能力的 Kubernetes 集群中的所有微服务治理平台组件，包括 Istio。根据需要，外部组件也可以集成，以实现容器化微服务治理。<br />- 管理员可以根据实际业务规模自定义网格组件的资源配置。                                                                                                         |
| **服务网格监控**                                   | 服务网格监控功能包括多个内置监控仪表板，从不同维度和粒度展示控制平面和数据平面的监控数据，使管理员可以实时查询或查看。                                                                                                                                                                                                                                                                                                                    |

| **追踪**                 | 该平台的追踪功能支持根据服务名称或 TraceID 查询当前命名空间中服务之间的调用链，监控调用链中服务的调用状态和持续时间等关键指标。结合详细的日志数据，帮助开发者了解业务请求链中每个节点或特定 API 的响应，快速定位和解决问题。<br /><br />- 方便开发人员和运维人员对服务之间的调用进行深入分析。<br />- 显示从调用链的开始时间和持续时间抽象出的散点图，以快速识别低性能链路。<br />- 支持全链路追踪（服务网格架构中的服务、Java 8 及以上服务）。<br />- 可通过 TraceID 快速定位调用链。<br />- 基于服务或 TraceID 查询链路信息。                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **服务治理**             | - 为不同类型的服务（服务网格架构中的服务、Java 服务）提供不同的治理能力。可以根据服务类型选择适当的治理方法。<br />- 使用可视化表单配置服务的治理策略，例如负载均衡、连接池管理、熔断和服务安全。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **服务路由**             | - 管理访问服务的所有流量的路由。<br />- 支持基于多组条件规则的流量路由管理，选项包括 uri、scheme、method、headers、port、sourceLabels 和各种匹配方式。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **服务网关**             | 服务网关，包括出口和入口网关，位于服务网格的边缘，管理进出服务网格的流量，允许您指定允许的流量。<br /><br />- 为服务网格配置多个入口或出口网关，每个网关允许通过多个可自定义的端口访问。<br />- 入口网关支持将外部流量根据各种路由场景路由到不同的后端资源（API 组、服务）；出口网关允许网格内的服务访问外部服务，而不影响开发体验，并对其流量进行监控。<br />- 支持多种使用场景，例如业务区域之间的网络隔离、特定业务的专用网关和端口隔离。<br />- 可以轻松停用不再需要的网关而不会造成损失。<br />- 基于 JWT 验证和授权客户端请求。<br />- 根据请求路径、请求头和 URI 重写配置网关路由规则。<br />- 支持 pod 级别的网关监控，内置多种告警策略，帮助用户及时发现和解决问题。监控数据包括 CPU、内存、QPS、连接数、进出流量指标。 |

## 服务发布管理

集成 Flagger，提供自动化的服务发布体验，支持 Canary、A/B 测试和蓝绿部署等多种部署策略。

| 特性                     | 描述                                                                                                                                                                                                                      |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **发布配置**             | 定义服务发布配置，以提供未来更改的模板化执行参数，实现对金丝雀版本 Pods 的自动部署和回滚。                                                                                                                             |
| **流量路由策略**         | 提供灵活的流量路由策略。服务可以根据发布配置模板自动路由流量，或为特定发布任务自定义流量路由规则。                                                                                                                        |
| **发布管理**             | 支持手动和自动两种发布选项。用户可以将发布决策委托给自动化系统，或选择手动发布。发布过程允许手动干预，并在任何时候可以回滚任务。                                                                                    |
| **可观测性**             | 提供主版本和金丝雀版本之间的指标比较和追踪追踪，实现金丝雀发布的质量分析和故障定位。                                                                                                                                          |

## 服务连接的可靠性治理

网格内的服务使用 HTTP/HTTP2 和 gRPC 等协议进行远程交互。当服务响应缓慢或不可用时，可能导致“级联故障”效应，占用前置服务的更多资源，并可能导致系统崩溃。

为避免此情况，必须建立保护机制和故障仿真以确保服务调用的可靠性，增强微服务系统的整体稳定性。

| 特性                          | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **流量策略**                   | 该平台提供统一的服务管理视角，允许用户通过可视化表单配置治理策略，例如负载均衡、连接池管理和服务熔断。<br />- **负载均衡策略**：根据指定的负载均衡策略，自动分配流量在服务 Pods 之间。<br />- **熔断策略**：检测异常服务实例，顺畅处理异常，自动隔离故障实例，并减少对下游服务的影响。<br />- **连接池设置策略**：通过配置适当的参数拒绝一些请求，以防止因流量过大导致系统崩溃。                                                                                                                                                                                                                    |
| **路由规则和策略**             | 支持在服务运行时动态配置故障恢复和故障注入策略，包括错误注入、延迟返回、超时重试、请求重写和流量镜像。这些策略有助于确保服务的可靠运行，防止局部故障向其他服务级联。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |

## 服务安全治理

在微服务治理场景中，服务调用从本地转向网络协议接口调用，带来了安全隐患。

| 特性                     | 描述                                                                                                                       |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **安全策略**             | 提供服务之间的流量加密。通过为服务设置安全策略，可以使用 mTLS 加密流量。                                          |
| **黑名单和白名单**       | 通过为服务设置黑名单或白名单，可以控制同一网格内服务之间的访问权限。                                             |

## 可视化服务运维

当服务故障发生时，常规的故障排除过程包括：跟踪异常流量 > 精确定位故障位置 > 缩小范围 > 分析具体原因。基于此过程，该平台提供完整的服务问题排查路径，帮助您基于异常流量迅速找到可疑位置并及时解决问题，尽量减少对业务的影响。

| 特性                                   | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **流量监控与 JVM 监控**                | 提供客户和服务器服务的多维流量监控数据面板，粒度细化到 API 级别；<br />- JVM 监控支持收集和显示 CPU/内存、线程和类等各个指标的监控数据。允许查看单个 pod 的数据或比较两个 pod 的数据，帮助开发者快速找到特定问题发生的指标项和时间点。                                                                                                                                                                                                                                                                                                                                                                                |
| **告警**                               | - 提供基于常见监控指标的告警功能，支持指标收集、策略管理、告警触发以及告警通知。结合流量监控和 JVM 监控，旨在提供完整的 **监控-告警-通知** 智能运维体验。<br />- 提供告警策略配置模板，支持单个告警策略配置或通过模板快速创建告警策略。<br />- 通过 **实时告警** 面板概览当前处于告警状态的资源信息，按告警级别过滤数据，查看告警详情，帮助用户了解问题的范围并快速定位根因。                                                                                                              |
| **实时日志**                           | 提供服务级别的实时日志面板，当发生服务异常时，允许您快速通过日志（包括访问日志）找到并排除问题。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **快速故障排查**                       | 当服务故障发生时，您可以结合丰富的可视化数据（服务拓扑、追踪）和监控与运维数据（监控、告警、日志）来跟踪和定位异常流量，快速解决问题。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

## 服务灾难恢复

在多区域、多数据中心场景中，当数据中心中的服务 pod 故障并被熔断隔离时，平台将自动将客户端的流量分配到其他数据中心的服务 pod，以确保服务稳定提供。

| 特性                               | 描述                                                                                                                                                                                                                     |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **就近路由**                        | 所有服务调用默认在所有数据中心使用就近路由。当调用链节点较长时，可以有效确保灾难恢复流量优先由同一集群的服务端点处理。                                                          |
| **自定义灾难恢复优先级**          | 根据集群所处区域配置区域负载，以自定义通过多个集群的灾难恢复流量的灾难恢复负载优先级。                                                                                                                                                    |
