# 介绍

## 服务网格 介绍

服务网格是基于 Istio 的 PaaS 平台，为应用程序和微服务提供云原生解决方案。 

该平台提供一键基础设施部署和升级能力、可视化的服务治理能力、高效的应用性能管理能力，以及高可用性和高性能的服务网关管理能力。它为正在进行数字化转型的企业提供了一整套稳定、可靠、开放的微服务解决方案。通过这些解决方案，企业可以提升服务治理效率，显著降低微服务框架的维护成本，减少开发人员对框架的依赖，从而使企业能够专注于业务发展，持续增强其核心竞争力。

## 服务网格 优势

服务网格的核心优势如下：

- 易用性

  通过可视化表单实现服务网格的一键部署和升级，无需开发成本即可体验丰富的非侵入式服务治理能力。

- 分布式微服务系统

  用分布式微服务替代传统单体架构，以满足现代业务对快速扩展、弹性资源分配、敏捷迭代和加速交付等需求。

- 云原生微服务生命周期管理

  无缝集成云原生生态系统（容器平台、DevOps），提供从开发/测试到部署/操作/治理的全生命周期管理，支持企业的容器化和微服务迁移。

- 简化的灰度发布

  基于动态流量策略实现服务版本间的自动（或受控）灰度发布。

- 全面可靠的服务治理

  平台提供可视化治理策略，包括动态路由配置、熔断、速率限制、实例级隔离和负载均衡规则，覆盖超过 90% 的微服务治理场景。

- 全球服务可观测性

  基于 OpenTelemetry 和 Istio，平台提供 360° 流量监控、分布式追踪和多维度分析能力，帮助快速优化和故障排查。

- 非侵入式数据平面扩展

  通过 WebAssembly (WASM) 插件扩展流量管理，管理 WASM 工件/配置作为 OCI 镜像，通过 ECDS (扩展配置发现服务) API 部署扩展并集中监控。

- 多区域灾难恢复

  高可用性架构具有独立集群、自动流量重定向和基于集群位置的定制故障转移优先级等特点，确保单个集群的故障不会影响其他集群。

- 灵活的多网关管理

  支持零停机时间部署，管理跨端口的网关，提供外部服务/API 分组和路由的可视化控制。

- 端到端运维系统

  集成监控服务/API 流量指标、JVM 监控和警报，提供统一日志的端到端收集、分析和搜索能力，加速故障排查。

## 服务网格 应用场景

服务网格的主要应用场景如下：

- 构建分布式微服务系统

  在将企业应用迁移到容器化和微服务的过程中，形成一个大型和复杂的分布式微服务架构。需要一个系统来管理和操作这些服务，以帮助开发人员解决服务发现、负载均衡、故障恢复、指标收集和监控、灰度发布、速率限制、访问控制、端到端认证等挑战。

- 服务全球可视化

  在复杂的微服务应用场景中，面对大量服务和复杂的服务间调用关系，开发人员需要实时监控服务的整体运行，理解组件间的通信。当业务故障或性能问题发生时，能够快速定位问题的具体位置，从而提高运营效率，降低业务运行风险。

- 服务发布管理

  在微服务应用的快速迭代和优化过程中，如果直接将新版本发布给所有用户，遇到在线事故（例如因大规模流量变化造成的故障、崩溃等），若未能快速解决问题，将严重影响用户体验。因此，在新版本发布时需要最小化对业务的影响，同时帮助开发人员验证新版本的功能、性能和用户满意度。

- 流量故障排查

  在大规模微服务系统中，服务间的调用关系复杂。一旦某个服务出错且故障持续较长时间，可能会级联影响其他服务，导致系统崩溃。因此，开发人员和微服务系统的运维人员需要实时观察服务健康状态，以防范潜在风险，并在遇到业务故障时借助详细的监控数据快速排查故障。

- 可靠的服务连接治理

  在微服务架构中，一个应用由多个服务构成，每个服务执行单一的业务功能。服务间的数据交互通过远程调用完成。为避免当服务 A 调用服务 B 时，若服务 B 调用其他服务，导致响应时间过长或其他服务不可用，从而使服务 A 的调用占用越来越多资源的问题，需实施流量治理策略和路由规则，确保服务连接的可靠性，增强微服务系统的整体稳定性。

- 安全的服务连接治理

  拆分单体应用为微服务后，提高了开发效率、系统稳定性和运行效率。因此，服务间的调用从本地调用转变为网络协议接口调用，引入了安全风险。因此，流量加密和服务间调用的授权配置，以及提供验证和审计功能，都是微服务治理平台中必不可少的。

- 服务网格生命周期管理

  完整的服务网格不仅包含 Istio 组件，还集成扩展组件。在多集群环境下，服务网格的管理变得尤为复杂。在服务网格的整个生命周期中，不仅涉及服务网格的部署、更新和删除，更重要的是，运维人员需要监控服务网格的健康状态，以便及时发现异常、排查故障，并确保业务的持续稳定支持。
