# 核心概念

## 监控数据

监控数据是指由系统及其行为发出的数据，包括追踪、指标和日志。

## 追踪

**追踪**记录了请求的路径（无论是来自应用程序还是终端用户）在多服务架构中传播的过程（如微服务和无服务器应用程序）。

一个追踪由一个或多个跨度组成。第一个跨度被称为根跨度，表示请求从开始到结束的整个生命周期。位于根跨度下的子跨度提供了关于请求过程的更详细的背景信息（或组成请求的各个步骤）。

没有追踪，识别分布式系统中性能问题的根本原因将会非常困难。追踪通过分解请求在系统中的传播流，使调试和理解分布式系统变得更容易。

## 跨度

**跨度**是分布式追踪的基本构建块，代表一个特定的操作或工作单元。每个跨度记录请求中的特定操作，帮助我们理解操作执行过程中的细节。

一个跨度包含名称、时间相关数据、结构化日志消息以及其他元数据（属性），这些信息共同展示了操作的完整情况。

根据跨度在追踪中的位置，可以区分出两种特殊类型的跨度：

- **根跨度**: 根跨度（追踪根跨度）是追踪中的第一个跨度（位于顶部），标志着请求的起始点。
- **服务入口跨度**: 服务入口跨度是服务作为服务器被调用时生成的第一个跨度，即请求进入服务时的跨度。

## 跨度标签

在分布式系统中，**跨度标签**是附加在每个跨度上的键值对信息，提供操作细节、资源标识符、性能指标和错误信息。它们为理解和分析服务性能及行为提供了必要的上下文，是调试和优化分布式应用程序的重要工具。

## 操作名称

**操作名称**是跨度的一个属性，指定了跨度内特定操作的名称。这个名称通常用于标识由跨度表示的特定业务逻辑或系统调用，如 HTTP 请求的方法加 URL 路径、数据库查询的简要描述或内部函数调用的名称。

## Jaeger

**Jaeger**是一个开源的**分布式追踪系统**。它旨在基于微服务架构监控和诊断复杂的分布式系统，帮助开发人员可视化请求追踪，分析性能瓶颈并排除异常。Jaeger 兼容**OpenTracing**标准（现为 OpenTelemetry 的一部分），支持多种编程语言和存储后端，是云原生领域的一个关键可观察性工具。

## OpenTelemetry 汇聚器

OpenTelemetry 汇聚器是一个与供应商无关的代理，可以接收、处理和导出监控数据。它支持以多种格式接收监控数据（例如 OTLP、Jaeger、Prometheus 以及许多商业/专有工具），并将这些数据发送到一个或多个后端。它还支持在导出之前处理和过滤监控数据。

要获取更多信息，请参见 [Collector](https://opentelemetry.io/docs/collector/)。
