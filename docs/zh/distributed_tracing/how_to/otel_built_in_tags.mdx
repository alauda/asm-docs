# 在追踪中使用 OpenTelemetry 服务内置标签

## 功能概述

Span 标签是指附加到 Span 的键值对数据，提供特定的上下文信息，以帮助开发人员更容易地理解和分析每个操作的特征和影响。

标签由 `key` 和 `value` 组成，其中 `key` 必须是字符串类型，而 `value` 可以是字符串、布尔值或数字。

## 标签类别

### 元数据

| Key                      | 描述                                                                                                                                                                                                                                                                                                                                                                               | 示例值       |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ |
| **internal.span.format** | 内部 Span 的数据格式。                                                                                                                                                                                                                                                                                                                                                             | `otlp`       |
| **otel.status\_code**    | Span 的状态。                                                                                                                                                                                                                                                                                                                                                                        | `ERROR`      |
| **span.kind**            | Span 的类型：<br />- **client**：表示该 Span 是一次外部调用（客户端操作），即客户端向服务或资源发起了请求。<br />- **server**：表示该 Span 是一次内部调用（服务器操作），即服务器接收并处理了来自客户端的请求。<br />- **internal**：表示应用程序内部的操作，不涉及与远程父或子节点的操作。 | `client`     |

### HTTP 请求信息

| Key                                | 描述                                                                                                                                                                | 示例值                          |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------- |
| **http.flavor**                    | HTTP 协议的版本。                                                                                                                                                    | `1.1`                          |
| **http.method**                    | HTTP 请求方法。                                                                                                                                                     | `GET`                          |
| **http.response\_content\_length** | HTTP 请求头中的一个字段，指定请求或响应体的长度（以字节为单位）。                                                                                                   | `12`                           |
| **http.route**                     | API。                                                                                                                                                               | `/get/{code}`                 |
| **http.scheme**                    | HTTP 协议的类型。                                                                                                                                                   | `http`                         |
| **http.status\_code**              | HTTP 状态码。                                                                                                                                                      | `200`                          |
| **http.target**                    | HTTP 请求的目标路径，通常包括资源的路径和可能的查询参数。                                                                                                         | `/get/200?foo=bar`            |
| **http.url**                       | 请求的 URL。<br />**注意**：仅对 `client` Span 类型可见。                                                                                                         | `http://calendar-app:9090/calendar` |
| **http.user\_agent**               | User-Agent 请求头，是一个特征字符串，有助于服务器和同伴识别请求用户代理的应用程序、操作系统、供应商和版本。                                                           | `okhttp/4.10.0`               |
| **upstreamURLPath**                | 上游服务的 URL 路径，指定应由上游服务处理的资源路径，包括可能的查询参数。                                                                                         | `/get/200?foo=bar`            |

| **net.host.port**             | 网络主机端口。                                 | `9090`              |

| ------------------------- | ---------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------- |

| **net.peer.port**         | 中间件服务的端口。                                                                                                                                                                | `5432`                                        |
| **net.sock.peer.name**    | Redis 服务的名称。                                                                                                                                                                 | `redis`                                       |
