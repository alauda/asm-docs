# Введение

## Введение в механизмы расширения Istio

Istio предоставляет возможности расширения, которые позволяют глубоко интегрироваться с плоскостью данных Envoy, сохраняя при этом целостность абстракции своей плоскости управления. Эти механизмы расширения дают возможность операторам платформы реализовывать сложные сценарии настройки, выходящие за рамки стандартных API Istio, особенно с помощью ресурса **EnvoyFilter** - интерфейса расширения в современных реализациях Istio.

:::tip Основной принцип проектирования
Расширения Istio действуют на основе принципа *контролируемой настройки*, позволяя изменения на уровне прокси при сохранении совместимости с декларативной моделью конфигурации Istio. Это балансирует гибкость и эксплуатационную безопасность.
:::

## Ключевые возможности расширения

### EnvoyFilter

Как основной механизм расширения Istio, EnvoyFilter позволяет напрямую управлять конфигурациями прокси Envoy через следующие ключевые операции:

- **Внедрение фильтров**\
  Вставить пользовательские L4/L7 фильтры в цепочку фильтров Envoy
- **Настройка протокола**\
  Реализовать обработку нестандартных протоколов с помощью пользовательских кодеков
- **Настройка параметров во время выполнения**\
  Переопределить настройки по умолчанию Envoy для конкретных рабочих нагрузок
- **Динамическая конфигурация**\
  Изменение слушателей, кластеров и маршрутов во время работы
