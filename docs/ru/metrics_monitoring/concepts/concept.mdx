# Основные концепции

## Трафик

Поток данных через сеть, систему или приложение. Эти данные могут включать запросы, сгенерированные пользователями, ответы сервисов или пакеты данных, охватывающие различные типы потоков данных в области информационных технологий и сетевых сред.

## Метрики мониторинга

Количественные данные, используемые для измерения и оценки производительности и состояния систем, сетей или сервисов. Эти метрики являются основными элементами в операциях ИТ и управлении системами, используемыми для постоянного мониторинга состояния системы, производительности и использования ресурсов, чтобы обеспечить стабильность сервиса и оптимизировать пользовательский опыт.

## Мониторинг API

Непрерывный процесс проверки и оценки производительности и отзывчивости микросервисов API (интерфейсов программирования приложений), наблюдая за тем, соответствуют ли API установленным стандартам производительности.

## Мониторинг JVM

Процесс отслеживания и анализа производительности и использования ресурсов экземпляров JVM, работающих с Java-приложениями. JVM — это среда, которая выполняет Java-программы, позволяя им работать на различных аппаратных и программных платформах без необходимости перекомпиляции. Мониторинг JVM имеет решающее значение для обеспечения эффективной и стабильной работы Java-приложений.

## RPS

Сокращение для Запросов в секунду, ключевой метрики для измерения производительности системы, сервера или приложения, указывая, сколько запросов система может обработать за секунду.

## TP50

Также известен как P50 или медиана. Эта метрика производительности измеряет середину набора данных. Например, в контексте времен отклика сети или времени обработки сервера **TP50** указывает на то, что 50% времени отклика запросов ниже или равны этому значению, представляя медианное время отклика. Общие процентные величины, используемые в мониторинге производительности и статистическом анализе, включают TP90 или TP99, которые представляют 90-й и 99-й перцентили соответственно. Эти метрики помогают администраторам понять производительность системы в различных условиях нагрузки или стресса.
