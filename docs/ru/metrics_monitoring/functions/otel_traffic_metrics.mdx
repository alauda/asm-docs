# Метрики Трафика OpenTelemetry

Платформа предлагает обширные данные метрик трафика, позволяя пользователям анализировать качество трафика по различным параметрам.

## Мониторинг Трафика Сервисов

### Просмотр Качества Трафика Сервиса

**Предварительные условия**

Сервис был внедрён с помощью Java Agent, пожалуйста, обратитесь к [Добавлению Сервисов](../../microservice/add-service.md#add-opentelemetry-service) для реализации.

**Шаги**

1. В левом навигационном меню нажмите **Мониторинг**.

2. Выберите сервис и временной диапазон, для которого вы хотите просмотреть данные мониторинга.

   **Примечание**: Временной диапазон запроса ограничен периодом хранения данных мониторинга Prometheus. Например, если данные мониторинга хранятся до 7 дней, а установленный временной диапазон составляет 30 дней, статистические данные будут охватывать только 7 дней.

3. Нажмите на соответствующие вкладки, чтобы просмотреть данные мониторинга трафика сервиса и данные мониторинга JVM.

### Обычные Операции

- **Обновить Данные**: Статистика мониторинга на текущей странице автоматически обновляется только один раз при открытии страницы. Чтобы обновить еще раз, вы можете использовать два следующих метода:

  - Ручное обновление: Нажмите <img src="../../../en/metrics_monitoring/assets/metric_refresh.png" style={{display: "inline", verticalAlign: "middle"}} /> в нижнем правом углу страницы, чтобы вручную обновить данные.

  - Установить Автообновление (по умолчанию выключено): Нажмите <img src="../../../en/metrics_monitoring/assets/metric_auto_refresh.png" style={{display: "inline", verticalAlign: "middle"}} /> для установки интервала автообновления данных.

- **Просмотреть/Установить Легенду**: Нажмите <img src="../../../en/metrics_monitoring/assets/legend.png" style={{display: "inline", verticalAlign: "middle"}} /> в верхнем правом углу графика, чтобы развернуть легенду в графике мониторинга. Нажмите на легенду, чтобы скрыть/показать соответствующую кривую на графике.

- **Просмотреть График Мониторинга В Большом Формате**: Нажмите <img src="../../../en/metrics_monitoring/assets/big_chart.png" style={{display: "inline", verticalAlign: "middle"}} /> в верхнем правом углу графика, чтобы в диалоговом окне с всплывающим окном увидеть увеличенный, более детализированный график мониторинга.

### Объяснение Мониторинговых Данных

| Мониторинговые Метрики           | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Среднее Время Ответа**         | Среднее время ответа в пределах временного диапазона запроса (общее время ответа / общее количество ответов).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Среднее Число Входящих/Исходящих RPS** | Среднее значение входящих/исходящих RPS (Запросов в секунду) в пределах временного диапазона запроса.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Время Ответа**                 | Время ответа между сервисами или внутри самого сервиса, отображается как среднее, TP 50, TP 95, TP 99.<br />TP (Топ Процентиль) xx указывает минимальную продолжительность, необходимую для xx процентов сетевых запросов, обычно используется в сценариях мониторинга производительности системы.<br />Наведите курсор на кривую, чтобы увидеть время ответа за конкретный период.                                                                                                                                                                                                                                                                                                                                    |
| **Входящие/Исходящие RPS**       | Общее количество входящих/исходящих RPS и количество ошибочных входящих/исходящих RPS в пределах временного диапазона запроса.<br />**RPS = Количество запросов в течение временного диапазона запроса / Продолжительность запроса (с)**.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Входящий/Исходящий Трафик**    | Объем входящих/исходящих запросов в пределах временного диапазона запроса и доля трафика по HTTP-кодам ответа (нормальные/2xx, 3xx, 4xx, 5xx).<br />Наведите курсор на столбчатую диаграмму, чтобы увидеть долю трафика по каждой категории.                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **Сравнение Клиентского Трафика**| **Клиент** относится к клиентскому сервису (служба нижнего уровня), который отправляет запросы к текущему сервису (служба верхнего уровня). **Сравнение Клиентского Трафика** будет сравнивать и отображать **Время Ответа**, **Входящие RPS** и входную **Ставку Ошибок RPS** (Ошибочные RPS/Общие RPS) различных клиентов, вызывающих текущий сервис.<br /><br />**Объяснение**:<br />- **неизвестный клиент** относится к группе клиентов (протокол HTTP) внутри сервисной сети, которые вызывают текущий сервис и чей трафик не управляется OpenTelemetry.<br />- Если клиент принадлежит к текущему пространству имен, нажатие на **Трассировку** рядом с именем клиента перенаправит вас на страницу трассировки. |
