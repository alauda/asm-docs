# Введение

## Мониторинг метрик

Платформа предоставляет комплексный мониторинг трафика, позволяя пользователям анализировать производительность и качество сервиса с различных точек зрения.

Кроме того, платформа предлагает мониторинг JVM, отображая метрики JVM (Java Virtual Machine), собранные с помощью OpenTelemetry Java Agent. Метрики включают сборку мусора (GC), кучу памяти, память вне кучи, потоки JVM, классы Java и т. д. Вы можете использовать данные мониторинга в реальном времени, представленные в графиках, для анализа производительности, быстро идентифицируя утечки памяти, аномалии потоков и другие проблемы.

## Уведомления

В контексте облачно-ориентированных технологий уведомления являются ключевым механизмом мониторинга, который своевременно информирует операционный персонал или разработчиков о потенциальных проблемах в системе. Этот механизм помогает командам быстро реагировать и разрешать проблемы, которые могут повлиять на производительность, доступность или безопасность приложения. Облачная среда обычно динамична и распределена, содержащая большое количество микросервисов и контейнеров, что делает эффективную систему уведомлений критически важным компонентом для поддержания стабильности системы.
