# Глубокое понимание типов узлов

:::note

В диаграмме топологии, если выявлен конфликт вызовов сервисов, не относящийся к вышеуказанным четким классификациям, он будет представлен иконкой узла **Неизвестный сервис**, чтобы обеспечить целостность и ясность отношений вызовов в топологии. Это представление помогает пользователям всесторонне понять структуру всей сети сервисов.

:::

## Типы узлов Istio

При управлении с помощью Service Mesh текущая платформа поддерживает следующие типы узлов:

1. **Сервис ServiceMesh**

2. **Шлюз Ingress**

3. **Шлюз Egress**

4. **Внешние сервисы (ServiceEntry)**

5. **PassthroughCluster** - Для получения дополнительной информации, пожалуйста, обратитесь к [официальной документации](https://istio.io/v1.20/blog/2019/monitoring-external-service-traffic/).

6. **BlackHoleCluster** - Для получения дополнительной информации, пожалуйста, обратитесь к [официальной документации](https://istio.io/v1.20/blog/2019/monitoring-external-service-traffic/).

## Типы узлов OpenTelemetry

При управлении с помощью OpenTelemetry текущая платформа поддерживает следующие типы узлов:

1. **Сервис OpenTelemetry**

2. **Промежуточное ПО**

- **Базы данных**
  - Oracle
  - Redis
  - MySQL
  - PostgreSQL
- **Очереди сообщений**
  - Kafka
  - RabbitMQ
  - RocketMQ
