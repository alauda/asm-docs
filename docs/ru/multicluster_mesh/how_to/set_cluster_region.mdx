# Как установить регион для кластера

:::tip
Как правило, кластеры в публичном облаке имеют атрибут региона. Этот документ объясняет, как установить атрибут региона для самостоятельно построенных кластеров на платформе или кластеров частного развертывания, подключенных к платформе.
:::

{/* TODO: [Установить регион для кластера](@console-platform-docs/zh/platform-usermanual/clusterall/1cluster/13clustermanagement/#region) */}

- Если вам нужно установить регион для **кластера, который будет добавлен** в mesh, обратитесь к [Установить регион для кластера]().

- Если вам нужно установить регион для **кластера, уже добавленного** в mesh, выполните следующие шаги:

  {/* TODO: @console-platform-docs/zh/platform-usermanual/clusterall/1cluster/13clustermanagement/#region */}

  1. Выполните операцию [Установить регион для кластера]().

  2. Войдите в любой управляющий узел кластера и выполните следующую команду для перезапуска компонента Istiod, развернутого в кластере:

     ```sh
     # Если номер версии Istiod равен 1.18, то имя развертывания Istiod по умолчанию - “istiod-1-18”; Istiod по умолчанию развернут в пространстве имен istio-system.
     kubectl rollout restart deployment <Имя развертывания Istiod> --namespace=<Пространство имен Istiod>
     ```

  3. Выполните следующую команду для перезапуска всех сервисов в кластере, в которые был внедрен Sidecar:

     ```sh
     kubectl rollout restart deployment <Имя развертывания сервиса> --namespace=<Пространство имен сервиса>
     ```
