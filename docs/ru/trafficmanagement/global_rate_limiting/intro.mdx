# Введение

## Глобальное ограничение скорости для сервисов

Модуль глобального ограничения скорости защищает сервисы от перегрузок в сценариях с высокой конкурентностью за счет интеллектуального управления трафиком. Он обеспечивает:

- Предотвращение сбоев сервиса во время пикового трафика
- Тонкое управление доступом с помощью многомерных правил
- Безопасное планирование емкости через анализ в теневом режиме

## Преимущества

- **Защита от высокой конкурентности**: Поддержание стабильности сервиса в пределах ресурсов
- **Точное управление**: Ограничение скорости на основе пути/метода/api-заголовка
- **Тестирование без риска**: Симуляция в теневом режиме без влияния на продукцию
- **Мониторинг в реальном времени**: Визуальное отслеживание эффектов ограничения

## Сценарии

- **Защита от перегрузки API**
  Защита критически важных API от резких увеличений трафика

- **Ограничение чувствительных операций**
  Принуждение к строгим пределам на операции записи/удаления

- **Управление квотами для нескольких арендаторов**
  Реализация справедливых политик использования среди групп клиентов

## Ограничения

- Требуется преднастроенная интеграция с кластером Redis
- Обязательная инъекция сайдкаров для целевых сервисов
- Максимум 1 глобальная политика на сервис
- Поддержка регулярных выражений только для синтаксиса RE2
- Период хранения данных в теневом режиме: 7 дней
