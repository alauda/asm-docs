# Введение

## Введение во внешние службы

Модуль внешних служб обеспечивает безопасное и управляемое соединение между службами внутри сервиса и внешними конечными точками. Эта возможность решает три фундаментальные задачи:

1. **Интеграция гибридной среды**: Соединяет облачные и устаревшие системы
2. **Управление трафиком**: Применяет политики сервисной сетки к не-сетевым службам
3. **Обеспечение безопасности**: Расширяет принципы нулевого доверия на внешние зависимости

Ключевые ценностные предложения:

- Снижение времени настройки соединений между средами на 80%
- Единая наблюдаемость для гибридных архитектур
- Последовательное применение политик в сетевых и не сетевых службах

## Преимущества

- **Гибкость протоколов**: Поддерживает протоколы L4 (TCP) и L7 (HTTP/HTTP2/HTTPS)
- **Динамическое балансирование нагрузки**: 4 интеллектуальных алгоритма с сохранением сессий
- **Инжиниринг устойчивости**: Прерывание цепи с механизмами автоматического восстановления
- **Контроль выхода**: Интеграция с шлюзом выхода Istio для безопасных выходных точек

## Сценарии

- **Миграция в облако**: Безопасное подключение рабочих нагрузок Kubernetes к локальным системам
- **Интеграция SaaS**: Управление трафиком к внешним API (Salesforce, Twilio и др.)
- **Маршрутизация между кластерами**: Реализация резервирования между облачными провайдерами
- **Модернизация устаревших систем**: Применение функций сервисной сетки к службам на базе виртуальных машин

## Ограничения

- Службы L4 не могут использовать функции сохранения сессий
- Выбор протокола не может быть изменен после создания
- Максимум 5 портов на определение службы
- Шлюз выхода требует ручного управления сертификатами
