# Введение

## Обзор модуля управления трафиком

Расширенные операции управления трафиком предоставляют комплексные возможности управления трафиком и бизнес-управления для облачно-ориентированных приложений. Этот модуль позволяет тонко контролировать коммуникацию сервисов с помощью таких функций, как передача трафика, автоматическое отключение, канарейковые релизы и глобальное ограничение скорости. Он поддерживает управление как входящим, так и исходящим трафиком, при этом соблюдая высокие стандарты наблюдаемости и безопасности в многокластерной среде.

## Преимущества

Основные преимущества расширенных операций управления трафиком:

- **Гибкое управление трафиком**\
Поддерживает конфигурацию передачи трафика и режим обхода, чтобы оптимизировать цепочки вызовов сервисов, сохраняя при этом возможности управления.

- **Комплексное управление сервисами**\
Предоставляет политику автоматического отключения, стратегии балансировки нагрузки и управление пулом соединений для обеспечения стабильности сервиса при высокой конкурентности.

- **Управление релизами в нескольких сценариях**\
Позволяет вручную/автоматически проводить канарейковые релизы с визуальным мониторингом, поддерживая проекты с синими-зелеными развертываниями и сценариями A/B тестирования.

- **Поддержка гибридной среды**\
Управляет внешними сервисами через шлюзы выхода и интегрирует несетевые сервисы через конфигурации внешних сервисов.

- **Увеличенная наблюдаемость**\
Предлагает мониторинг в реальном времени доступов, эффектов ограничения скорости и состояния здоровья Pods через визуальные панели мониторинга.

## Сценарии

Основные сценарии применения включают:

- **Оптимизация производительности**\
Используйте передачу трафика для промежуточных сервисов, чтобы обойти ненужную обработку Sidecar.

- **Быстрая диагностика ошибок**\
Включите режим обхода, чтобы проверить, вызваны ли аномалии сервиса прокси-сервисами Sidecar.

- **Управление версиями микросервисов**\
Реализуйте канарейковые релизы для безопасных производственных развертываний новых версий сервиса.

- **Защита при высокой конкурентности**\
Конфигурируйте глобальное ограничение скорости и автоматическое отключение, чтобы предотвратить лавины сервисов.

- **Управление гибридной архитектурой**\
Подключайте традиционные сервисы и облачно-ориентированные сервисы через регистрацию внешних сервисов.

- **Контроль сетевой безопасности**\
Быстро изолируйте аномальные Pods, сохраняя нормальную работу сервисов.

## Ограничения

1. Требуется инъекция Sidecar для полноценной функциональности (за исключением внешних сервисов)
2. Канарейковый релиз не может сосуществовать с существующими конфигурациями маршрутизации сервисов
3. Статус изоляции Pods сбрасывается во время операций развертывания/масштабирования
4. Глобальное ограничение скорости требует конфигурации кластера Redis
5. Ограничения на обнаружение сервисов между кластерами для канарейковых релизов
6. Режим обхода требует кластеров Kubernetes версии v1.23+
7. Конфигурация шлюза выхода требует региональной согласованности в многокластерных сценариях
8. Политики не применяются к сервисам, настроенным на передачу трафика
