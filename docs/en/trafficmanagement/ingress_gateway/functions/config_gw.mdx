# Gateway Configuration

## Introduction
Gateway Configuration defines logical gateways with network exposure parameters to:

- Establish standardized ingress points for service mesh
- Enable protocol-aware traffic listening
- Integrate with existing DNS/certificate infrastructure
- Facilitate traffic routing rule binding

Core value: Unified entry management for north-south traffic

## Features
- Multi-protocol port definitions (HTTP/HTTPS/TCP)
- Host-based traffic filtering
- SNI configuration for TLS termination
- DNS integration with platform-managed domains

## Advantages
**Security**: Certificate enforcement for HTTPS
**Flexibility**: Wildcard host support (`*`)
**Compliance**: 2048-bit minimum key length requirement
**Observability**: Built-in integration with routing metrics

## Creating Gateway Configuration

### Step 1: Access Configuration Interface
1. Navigate: **Service Mesh** > **Gateway**
2. Select target ingress gateway
3. Open **Gateway Configuration** tab
4. Click **Create Gateway Configuration**

### Step 2: Core Parameter Setup
```yaml
apiVersion: networking.istio.io/v1beta1
kind: Gateway
metadata:
  name: main-gateway
spec:
  servers:
  • port:
      number: 443
      protocol: HTTPS
    hosts:
    ◦ "www.example.com"
```

| Parameter | Required | Constraints |
|-----------|----------|-------------|
| Namespace | Yes | Predefined gateway location |
| Port | Yes | Must match gateway deployment |
| Hosts | Yes | Valid domain or `*` wildcard |

### Step 3: Protocol-Specific Rules
**HTTPS Requirements**:
```bash
# Certificate validation check
openssl x509 -text -in example.crt | grep 'Key Usage' -A1
```
- Must use platform-registered domains
- Certificate key length ≥2048 bits

**TCP Limitations**:
- Only supports wildcard host (`*`)
- No TLS termination capability

## Configuration Management

### Post-Creation Operations
1. Project administrators can:
   - View gateway configurations
   - Create associated routing rules
   - Monitor traffic metrics
2. DNS verification:
```bash
dig +short <configured-domain> @8.8.8.8
```

### Update Constraints
- Port protocol cannot be modified after creation
- Host changes require certificate revalidation
- Namespace binding is immutable
